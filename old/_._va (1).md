## 2. 스프린트 2 (개발 1주차) — 기획회의 필수 안건

> 코딩 시작 전 반드시 완료. 미결 시 개발 중 충돌 발생.

| 안건 | 내용 | 긴급도 |
|------|------|--------|
| **A. 단위 변환 상수** | 1 unit = 1mm vs 100mm → geometry_utils.js MM_TO_UNIT 고정 | **치명적** |
| **B. calculate_position 명세** | 방향별 오프셋 계산 규칙 확정 | **치명적** |
| **C. 증분 검증 파라미터** | step_mm / max_steps 확정 | **높음** |
| **D. Agent 3 재배치 범위** | 재호출 후 실패 오브젝트만 vs 전체 재시작 | **높음** |
| 서비스 이해도 통일 | Agent 1~3 / dict 구조 / 증분 검증 / Circuit Breaker / 마킹 흐름 | 높음 |
| 배치 실패 부분 출력 기준 | 몇 개 이상 성공 시 출력할 것인가 | 높음 |
| 역할 분담 | Agent별·모듈별 담당자 확정 | 중간 |

### C. 증분 검증 파라미터 상세

```
step_mm: 50 / 100 / 200mm
max_steps: 5 / 10 / 20

트레이드오프:
  step_mm 작고 max_steps 클수록 → 성공률 높음 / 연산 시간 증가
  step_mm 크고 max_steps 작을수록 → 빠름 / 조정 실패율 증가

스프린트 3에서 실측 후 스프린트 5에서 재조정 가능
```

### D. Agent 3 재배치 범위

```
선택지 A: 실패 오브젝트만 재배치
  장점: 기배치 유지, API 호출 최소
  단점: 새 배치가 기존과 충돌 가능성

선택지 B: 전체 재시작
  장점: 일관성 보장
  단점: 비용 증가, 기존 성공 결과 폐기

→ 스프린트 2 기획회의에서 확정
```

## 6. 데모 시나리오

```
카카오프렌즈 성수동 팝업스토어:

입력:
  도면: ㄱ자 (6000×4000mm + 4000×2000mm)
        스프링클러 2개, 소화전 미표기
  브랜드 메뉴얼: character_bbox clearspace 1500mm, 입구 정면

기대 흐름:
  입력 화면: 메뉴얼 업로드 → Agent 1 추출 → clearspace 1500mm 확인
  Agent 2 전반부: 스프링클러 감지, 소화전 미감지
  사용자 마킹: 소화전 위치 클릭 추가
  Agent 2 후반부: Dead Zone 3개 + reference_points + walk_distance 계산
  1단계 확인: Dead Zone + 배치 영역 시각화 → "진행"
  Agent 3: 자연어 판단 →
    character_bbox → entrance / inward / priority 1
    shelf_rental → north_wall_mid / wall_facing / priority 2
    photo_zone → inner_corner / entrance_facing / priority 3
  Shapely + 증분 NetworkX:
    character_bbox → 통과 → 확정
    shelf_rental → 통로 좁아짐 → 코드 조정 → 성공 → 확정
    photo_zone → 통과 → 확정
  검증: passed
  출력: Whitebox 3D + .glb + 템플릿 리포트

발표 포인트:
  LLM이 수치를 모르는 구조 (dict 분리)
  오브젝트 단위 증분 검증으로 데드락 차단
  코드 레벨 위치 조정 → Agent 3 재호출 없이 해결
  사용자 마킹으로 설비 미감지 대응
  면책 조항 자동 삽입
  Circuit Breaker로 JSON 파싱 오류 방어
```
