# 남은 작업 목록

## 심각: 동선(Main Artery) 시스템 미완성

현재 Main Artery는 **매장 동선 가이드로서 역할을 하지 못하는 상태**입니다.

### 문제 1: 단일 직선 경로 — 매장 절반 미커버

- **현상**: 입구(좌상단) → Dijkstra 최원점(우하단) 대각선 한 줄만 생성
- **결과**: 우측 상단·우측 중앙에 배치된 오브젝트가 동선 유도 범위 밖에 방치됨
- **원인**: `walk_mm_calculator.py` `_compute_main_artery()`가 입구↔최원점 최단경로 1개만 계산
- **필요한 것**: 입구 → 매장 전체를 순회 → 입구(또는 출구)로 돌아오는 루프형 경로
- **바닥 기하학만으로 가능**: 기물 종류를 몰라도, 바닥 폴리곤 꼭짓점 경유 순회 경로로 전체 면적 커버 가능

### 문제 2: 경로 계산 시 배치 오브젝트 미반영

- **현상**: Main Artery가 배치된 기물을 직선으로 뚫고 지나감
- **원인**: `corridor_graph.py` `build_corridor_graph()`가 `usable_poly`와 `dead_zones`만 장애물로 인식. 배치 완료된 오브젝트를 장애물로 등록하지 않음
- **경로 계산 시점**: Agent 2 단계(배치 전)에서 계산 → 배치 후 재계산 없음
- **해결 방향**: 배치 완료 후 오브젝트 footprint를 장애물로 추가한 그래프에서 Main Artery 재계산

### 문제 3: 3D 리본 깊이 버퍼 문제 (해결 완료)

- **현상**: 핑크색 Ribbon Mesh가 바닥에 묻혀 안 보임
- **원인**: 카메라 near=1 / far=100000 (비율 100,000:1) → 깊이 버퍼 해상도 ~13mm, 바닥(Y=5)과 리본(Y=8)의 3mm 차이 구분 불가
- **해결**: `logarithmicDepthBuffer: true` 적용 + near=10으로 상향. ROAD_Y=6으로 정상 동작
