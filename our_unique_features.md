# 템플릿 미포함 — Shin(우리)만 있는 기능

> Rendy 템플릿 기준 비교 시 항목 자체가 없어서 비교 불가한 기능들
> 2026-04-03

---

## 레퍼런스 이미지 검색 (Tavily)

- 브랜드 카테고리 기반으로 Tavily API에서 팝업스토어 조감도 이미지 검색
- 이미지 3장을 LLM Vision에 첨부하여 배치 컨셉 참고
- 쿼리: "aerial view top down layout 조감도" 고정 (매장 정면 사진 배제)
- **의도:** LLM이 텍스트만 보고 배치하면 기계적. 실제 팝업 사례를 "보고" 컨셉을 가져옴

## 가벽 자동 배치 (Step 0.5)

- 오브젝트 배치 전에 LLM이 가벽(파티션) 필요 여부 + 위치를 결정
- 가벽 배치 후 → dead_zone 갱신 → NetworkX 재생성 → 양면 reference_point 생성
- 가벽은 dead zone이 아닌 **오브젝트**로 구현 → 3D에서 이동/크기조절/회전 가능
- **의도:** 가벽이 공간을 나누면 새 벽면이 생기고, 그 위에 선반 등 배치 가능. 오브젝트보다 먼저 설치해야 파이프라인이 자연스러움

## placement_rules 하네스

- LLM이 배치 결정한 뒤 코드가 브랜드 규정을 강제 교정
- `preferred_wall`: LLM이 선택한 ref가 조건 안 맞으면 자동 교체 (예: facing_entrance 필수인데 side wall 선택 → 교체)
- `required_direction`: LLM 재량 없이 direction 고정
- `min_clearspace_mm`: 브랜드 이격 거리가 기본 gap보다 크면 브랜드 규정 우선
- **의도:** LLM은 자유롭게 결정하되, 브랜드 매뉴얼 위반은 코드가 사후에 잡아줌. "매뉴얼이 왕" 원칙 구현

## brand_category 자동 감지

- Agent1이 매뉴얼에서 브랜드 업종을 자동 분류 (8종: 캐릭터 IP / 패션 / F&B / 뷰티 / 테크 / 아트 / 엔터 / 기타)
- 마킹 단계에서 사용자가 드롭다운으로 수정 가능
- 레퍼런스 이미지 검색 쿼리에 사용
- **의도:** 카테고리별로 배치 패턴이 다름 (캐릭터 IP = 등신대 중심 vs F&B = 카운터 중심). 검색 쿼리 자동 최적화

## 벽별 수용량 계산 (compute_wall_capacities)

- 각 wall reference_point에 대해 오브젝트 타입별 최대 수용 수를 코드가 계산
- 공식: `usable_len = wall_length - clearance × 2` → `capacity = usable_len / (obj_width + gap)`
- 중앙/코너는 면적 기반 추정
- LLM에 "선반 최대 7개"로 전달 (mm 수치 비노출)
- **의도:** LLM에 수치를 주면 산수를 시도하고 틀림. 대신 코드가 상한을 계산하고 LLM은 범위 내에서 결정

## LLM temperature 제어

- Vision 감지: temperature=0 (일관된 결과 — 매번 같은 좌표)
- 배치 결정: temperature=0.7 (적절한 다양성 — 반려 후 재배치가 다르게 나옴)
- **의도:** 감지는 정확해야 하고, 배치는 다양해야 함. 용도별 온도 분리

## Agent1 + Agent2a 병렬 실행

- 브랜드 매뉴얼 추출(Agent1)과 도면 감지(Agent2a)는 의존성 없음 → Promise.all 병렬
- Agent2b(공간 계산)부터 합류
- **의도:** 응답 시간 단축. 직렬이면 매뉴얼 추출 5초 + 도면 감지 10초 = 15초. 병렬이면 max(5, 10) = 10초

## 도미노 검증 구간 (2곳)

- **검증 1 (Agent2→Agent3):** polygon ≥ 3점, 면적 ≥ 10㎡, ref_point ≥ 2개, wall ref ≥ 1개
- **검증 2 (Agent3 LLM 후):** 배치 0개면 경고, 활용률 5% 미만이면 경고
- **의도:** 도미노 실패 방지. polygon이 잘못됐으면 Agent3으로 넘기지 않고 조기 차단. 트러블슈팅에서 "하나 틀리면 전부 틀리는" 연쇄 실패를 경험하고 추가

## gap 기반 충돌 체크

- `attempt_placement`에서 오브젝트 polygon을 `buffer(gap_mm)` 확장 후 intersects 체크
- 기존: 오브젝트 경계가 안 닿으면 통과 (0mm 간격도 OK)
- 변경: gap_mm(300mm) 이내에 다른 오브젝트 있으면 거부
- **의도:** 오브젝트끼리 물리적으로 안 겹쳐도 "붙어있으면" 비현실적. 최소 간격 보장

## 반려 피드백 전달

- 사용자가 "이 배치 마음에 안 들어요" → 반려 사유(동선 불편/히어로 위치/공간 낭비 등) 선택
- 사유가 space_data에 `rejection_feedback`로 포함되어 Agent3 재배치 시 참고
- **의도:** 맹목적 재시도가 아닌, 사용자 피드백을 반영한 재배치. "동선 불편"이면 LLM이 동선 위주로 재설계

## "60~70% 활용" 프롬프트 가이드

- LLM에 "공간의 60~70%를 오브젝트로 활용하세요, 30~40%는 동선"
- 수용량 상한의 50~80% 권장
- **의도:** LLM이 보수적으로 선반 1개만 놓는 문제 해결. 코드가 상한을 주고 LLM이 범위 내에서 적극적으로 채우도록 유도
